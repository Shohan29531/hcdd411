Compass: Install, Open, and Connect

Install MongoDB Compass

macOS
  1. Download the Compass installer for macOS
    https://www.mongodb.com/try/download/compass
  2. Open the downloaded .dmg file
  3. Drag MongoDB Compass into Applications
  4. Open Compass from Applications
Windows
  1. Download the Compass installer for Windows
    https://www.mongodb.com/try/download/compass
  2. Run the installer and keep the normal default options
  3. Finish installation and open MongoDB Compass

Do this in: Compass GUI

  1. First make sure your local MongoDB server is already running
  2. Open Compass and go to the New Connection screen
  3. In the connection box, paste mongodb://localhost:27017
  4. Leave the local defaults alone unless your setup is different
  5. Click Connect or Save & Connect
  6. You should now see your local deployment in the left sidebar
Important: Compass is only the viewer/editor. If the MongoDB server is not running, Compass cannot connect.
Meaning of localhost:27017: same computer, default MongoDB port.

Compass: Where to Click to See Your Data

After Compass connects

  1. In the left sidebar, click your local deployment
  2. Click the database name, for example school
  3. Click the collection name, for example students
  4. Open the Documents tab to see the stored documents
  5. After you run commands in mongosh, click Refresh in Compass

What you should look for

Where in CompassWhat it shows
Left sidebarYour connection and database names
Database viewThe collections inside that database
Documents tabThe actual JSON-like documents
Refresh buttonAny inserts, updates, or deletes you just ran in mongosh
Best beginner workflow: keep Compass open on one side and mongosh on the other. Run a command, then refresh Compass so you can watch the data change.

Before We Start: Where Do Commands Go?

Run in: Terminal / Command Prompt / PowerShell

  • Install software
  • Start or stop MongoDB server
  • Check versions
  • Run mongosh

Run in: mongosh

  • Create databases and collections
  • Insert, read, update, delete documents
  • Run MongoDB commands like use school or db.students.find()

Do this in: Compass GUI

  • Create a connection
  • Browse collections visually
  • Insert/edit documents with forms
  • Run queries without typing everything manually

Do not mix them up

  • brew install ... does not go in mongosh
  • db.users.find() does not go in Terminal
  • Compass steps are clicks, not shell commands

We will do all CRUD examples in mongosh. You can later do many of the same actions in Compass with buttons and forms.

Create: Insert One Document

Run in: mongosh
▷ INSERTONE
> db.students.insertOne({ name: "Alice", age: 20, major: "HCDD" })

What changes after you run it

▷ RESULT + COMPASS AFTER REFRESH
{ acknowledged: true, insertedId: ObjectId("...") }
# Compass -> school -> students -> Documents
{ _id: ObjectId("..."), name: "Alice", age: 20, major: "HCDD" }
If you do not provide _id, MongoDB creates one automatically.

Create: Insert Many Documents

Run in: mongosh
▷ INSERTMANY
> db.students.insertMany([{ name: "Bob", age: 21, major: "IST" }, { name: "Carol", age: 19, major: "HCDD" }, { name: "David", age: 22, major: "CS" }])

What changes after you run it

▷ RESULT + COMPASS AFTER REFRESH
{ acknowledged: true, insertedIds: { 0: ObjectId("..."), 1: ObjectId("..."), 2: ObjectId("...") } }
# Compass now shows 4 documents total:
Alice, Bob, Carol, David

Read: Find Documents

Run in: mongosh
▷ BASIC READS
> db.students.find()
> db.students.findOne({ name: "Alice" })
> db.students.find({ major: "HCDD" })

What you should see

▷ MATCHING DOCUMENTS
> db.students.findOne({ name: "Alice" })
{ _id: ObjectId("..."), name: "Alice", age: 20, major: "HCDD" }
> db.students.find({ major: "HCDD" })
{ name: "Alice", age: 20, major: "HCDD" }
{ name: "Carol", age: 19, major: "HCDD" }
# Compass shows the same documents in the Documents tab
  • find() returns many matching documents
  • findOne() returns one matching document
  • The object inside find(...) is your filter

Read Operators: Comparison

Run in: mongosh
▷ COMPARISON QUERIES
> db.students.find({ age: { $gt: 20 } })
> db.students.find({ age: { $gte: 20 } })
> db.students.find({ age: { $lt: 21 } })
> db.students.find({ age: { $in: [19, 20] } })

Useful operators

  • $gt greater than
  • $gte greater than or equal
  • $lt less than
  • $lte less than or equal
  • $in value appears in a list

Read Operators: Logical + Projection + Sort

Run in: mongosh
▷ LOGICAL + OUTPUT CONTROL
> db.students.find({ $and: [{ major: "HCDD" }, { age: { $gte: 20 } }] })
> db.students.find({}, { name: 1, major: 1, _id: 0 })
> db.students.find().sort({ age: -1 })

Meaning

  • $and combines conditions
  • Projection chooses which fields appear
  • 1 means ascending, -1 means descending sort

Update: Modify One or Many Documents

Run in: mongosh
▷ UPDATEONE / UPDATEMANY
> db.students.updateOne({ name: "Alice" }, { $set: { age: 21 } })
> db.students.updateMany({ major: "HCDD" }, { $set: { active: true } })

What changes after refresh

▷ RESULT + COMPASS AFTER REFRESH
Alice now has age: 21
Alice and Carol now have active: true
{ name: "Alice", age: 21, major: "HCDD", active: true }
{ name: "Carol", age: 19, major: "HCDD", active: true }
  • First argument = which documents
  • Second argument = what change to apply

Update Operators You Will Use Most Often

Run in: mongosh
OperatorWhat it doesExample
$setSet a field to a value{ $set: { major: "DS" } }
$incIncrease a number{ $inc: { age: 1 } }
$unsetRemove a field{ $unset: { nickname: "" } }
$pushAdd one item to an array{ $push: { courses: "IST 256" } }

Delete: Remove Documents

Run in: mongosh
▷ DELETEONE / DELETEMANY
> db.students.deleteOne({ name: "Bob" })
> db.students.deleteMany({ active: true })

What disappears after refresh

▷ RESULT + COMPASS AFTER REFRESH
Bob is removed
Alice and Carol are removed because they had active: true
# Remaining document in Compass:
{ name: "David", age: 22, major: "CS" }
  • deleteOne() removes only the first matching document
  • deleteMany() removes all matching documents that match the filter
  • Always double-check the filter before running delete operations

Compass vs mongosh

Use Compass when…

  • You want to browse data visually
  • You are learning the shape of documents
  • You want forms instead of typing

Use mongosh when…

  • You want repeatable commands
  • You are following code examples from class
  • You need quick CRUD/testing from the terminal
Good workflow for beginners: explore in Compass, practice commands in mongosh.

Common Beginner Problems

Problem

  • mongosh not recognized
  • Compass cannot connect
  • Connection refused on localhost:27017
  • Started server in wrong shell or wrong command

Check first

  • Is MongoDB server actually running?
  • Did you install mongosh separately?
  • Are you using the correct connection string?
  • Did you reopen the terminal after installation?

Practice: Build a Tiny Student Roster DB

Run in: mongosh
▷ PRACTICE FLOW
> use roster
> db.createCollection("students")
> db.students.insertMany([{ name: "Ava", year: 2, major: "HCDD" }, { name: "Liam", year: 1, major: "IST" }])
> db.students.find()
> db.students.updateOne({ name: "Ava" }, { $set: { year: 3 } })
> db.students.deleteOne({ name: "Liam" })

Mongo Shell: Count + Sort

Run in: mongosh
▷ COUNT + SORT
> db.students.countDocuments()
> db.students.find().sort({ year: 1 })
> db.students.find().sort({ year: -1 })

What these do

  • countDocuments() gives the number of documents in the collection
  • sort({ year: 1 }) sorts ascending
  • sort({ year: -1 }) sorts descending

Example result

▷ SAMPLE OUTPUT
2
{ name: "Liam", year: 1, major: "IST" }
{ name: "Ava", year: 2, major: "HCDD" }

Mongo Shell: Skip + Limit

Run in: mongosh
▷ PAGINATION STYLE QUERIES
> db.students.find().sort({ name: 1 }).limit(2)
> db.students.find().sort({ name: 1 }).skip(2)
> db.students.find().sort({ name: 1 }).skip(2).limit(2)
Idea: limit() caps how many documents you get back. skip() ignores the first few matches. Together, they act like simple pagination.

Example

If the names are sorted as Ava, Bob, Carol, David, then skip(2).limit(2) returns Carol and David.

Indexes: Why They Matter

Without an index

  • MongoDB may scan many documents to find matches
  • That is fine for tiny class examples
  • It becomes slow on large collections

With an index

  • MongoDB keeps a structure that helps it find values faster
  • This improves query performance for common filters and sorts
  • Indexes cost extra storage and slightly slower writes
A good beginner mental model: an index is like the index at the back of a textbook. You do not scan every page from the beginning to find one topic.

Mongo Shell: getIndexes() + createIndex() + dropIndex()

Run in: mongosh
▷ INDEX COMMANDS
> db.students.getIndexes()
> db.students.createIndex({ major: 1 })
> db.students.getIndexes()
> db.students.dropIndex("major_1")

What to expect

  • Before you create one, you usually only see the default _id index
  • createIndex({ major: 1 }) creates an ascending index on major
  • The name is usually major_1

Good use case

If you often run db.students.find({ major: "HCDD" }), indexing major can make those lookups faster.

Takeaways

  • Use databases for persistence, querying, and shared access
  • SQL and NoSQL solve different kinds of problems
  • MongoDB fits well with JSON-heavy Node/Express apps
  • You need three separate pieces: server, mongosh, and Compass
  • Always know where a command should be run
