What is Mongoose?

The short answer

Mongoose is an Object Data Modeling (ODM) library that sits between your Node.js code and MongoDB.

  • Install via npm — it is a third-party package
  • Official docs: mongoosejs.com
  • The most widely-used Node.js + MongoDB pairing in production

What it gives you

  • Schema definitions — declare what fields a document must have
  • Built-in validation — required fields, type checks, custom rules
  • Model methods — find, create, findByIdAndUpdate, and more
  • Middleware (hooks) — run logic before or after save, delete, etc.
  • Clean Express integration — designed to work with the stack you already know
Think of Mongoose as adding structure and guardrails on top of MongoDB's flexibility. The next slide shows exactly what that means in real code.

Raw Driver vs Mongoose — In Code

Same task: insert a student. Look at what the raw driver silently allows vs. what Mongoose catches.

Raw MongoDB Driver

📄 raw-driver.js
  1. const { MongoClient } = require("mongodb");
  3. async function run() {
  4. const client = new MongoClient("mongodb://127.0.0.1:27017");
  5. await client.connect();
  6. const col = client.db("school").collection("students");
  8. // Works — correct data
  9. await col.insertOne({ name: "Alice", age: 20 });
  11. // Also works — age is missing. No error!
  12. await col.insertOne({ name: "Bob" });
  14. // Also works — completely wrong shape. No error!
  15. await col.insertOne({ foo: "bar" });
  17. await client.close();
  18. }
  19. run();
All three inserts succeed. Your collection now has garbage data and you will not find out until something breaks in your app later.

Mongoose

📄 mongoose-example.js
  1. const mongoose = require("mongoose");
  3. const studentSchema = new mongoose.Schema({
  4. name: { type: String, required: true },
  5. age: { type: Number, required: true },
  6. });
  7. const Student = mongoose.model("Student", studentSchema);
  9. await mongoose.connect("mongodb://127.0.0.1:27017/school");
  11. // Works — correct data
  12. await Student.create({ name: "Alice", age: 20 });
  14. // Throws — age is required
  15. await Student.create({ name: "Bob" });
  16. // ValidationError: age: Path `age` is required.
  18. // Throws — both name and age are required
  19. await Student.create({ foo: "bar" });
  20. // ValidationError: name: Path `name` is required.
Mongoose catches bad data at write time — before it reaches the database.

config/database.js

File: config/database.js
📄 config/database.js
  1. const mongoose = require("mongoose");
  3. const connectDB = async () => {
  4. try {
  5. await mongoose.connect("mongodb://127.0.0.1:27017/school");
  6. console.log("MongoDB connected successfully");
  7. } catch (error) {
  8. console.error("MongoDB connection failed:", error.message);
  9. process.exit(1); // stop the app — no point running without a DB
  10. }
  11. };
  13. module.exports = connectDB;

URL breakdown

  • mongodb:// — protocol
  • 127.0.0.1:27017 — localhost, MongoDB default port
  • school — the database name (auto-created on first write)

process.exit(1)

If the database is unreachable, the whole server should stop. Exiting with code 1 signals failure — useful in deployment pipelines and scripts that restart the process.

models/students.model.js — Schema & Model

File: models/students.model.js

The complete file

📄 models/students.model.js
  1. const mongoose = require("mongoose");
  3. const studentSchema = new mongoose.Schema(
  4. {
  5. // required: field must be present or Mongoose throws
  6. name: { type: String, required: true },
  8. // min: number cannot be below 0
  9. age: { type: Number, required: true, min: 0 },
  11. // default: value used when field is omitted
  12. major: { type: String, default: "Undeclared" },
  13. },
  14. { versionKey: false } // hides the __v field from output
  15. );
  17. // "Student" → lowercase → pluralize → "students" collection
  18. const Student = mongoose.model("Student", studentSchema);
  20. module.exports = Student;

Common schema options

OptionWhat it does
required: trueField must be in the document
default: valueUsed when the field is omitted
unique: trueNo two documents can share this value
min / maxNumber range constraint
minlength / maxlengthString length constraint
Auto-naming: "Student" → student → students.

Mongoose lowercases and pluralizes the model name to find the matching collection.

This is just the default. You can choose your own collection name by passing a third argument to mongoose.model(...), or by setting the collection option in the schema.

server.js — Complete Express + Mongoose App

File: server.js — copy this entire file
📄 server.js
  1. const express = require("express");
  2. const connectDB = require("./config/database");
  3. const Student = require("./models/students.model");
  5. const app = express();
  6. app.use(express.json()); // parse incoming JSON request bodies
  8. // ── GET /students ── return all students ──────────────────────
  9. app.get("/students", async (req, res) => {
  10. try {
  11. const students = await Student.find();
  12. res.json(students);
  13. } catch (err) {
  14. res.status(500).json({ error: err.message });
  15. }
  16. });
  18. // ── POST /students ── create a new student ────────────────────
  19. app.post("/students", async (req, res) => {
  20. try {
  21. const student = await Student.create(req.body);
  22. res.status(201).json(student);
  23. } catch (err) {
  24. res.status(400).json({ error: err.message });
  25. }
  26. });
  28. // ── PUT /students/:id ── update a student by ID ───────────────
  29. app.put("/students/:id", async (req, res) => {
  30. try {
  31. const student = await Student.findByIdAndUpdate(
  32. req.params.id,
  33. { $set: req.body },
  34. { new: true } // return the updated doc, not the old one
  35. );
  36. if (!student) return res.status(404).json({ error: "Not found" });
  37. res.json(student);
  38. } catch (err) {
  39. res.status(400).json({ error: err.message });
  40. }
  41. });
  43. // ── DELETE /students/:id ── delete a student by ID ───────────
  44. app.delete("/students/:id", async (req, res) => {
  45. try {
  46. const student = await Student.findByIdAndDelete(req.params.id);
  47. if (!student) return res.status(404).json({ error: "Not found" });
  48. res.json({ message: "Deleted successfully", student });
  49. } catch (err) {
  50. res.status(500).json({ error: err.message });
  51. }
  52. });
  54. // ── Start: connect DB first, then listen ──────────────────────
  55. async function startServer() {
  56. await connectDB();
  57. app.listen(3000, () => {
  58. console.log("Server running on http://localhost:3000");
  59. });
  60. }
  62. startServer();

Validation in Action

Try these intentionally bad requests. Read the errors — they come directly from the Mongoose schema.

Missing a required field

▶ TERMINAL — send without age
$ curl -X POST http://localhost:3000/students \
-H "Content-Type: application/json" \
-d '{"name": "Dan"}'
▶ RESPONSE — 400 Bad Request
{
"error": "Student validation failed:
age: Path `age` is required."
}

Wrong data type

▶ TERMINAL — send age as a string
$ curl -X POST http://localhost:3000/students \
-H "Content-Type: application/json" \
-d '{"name": "Eve", "age": "twenty"}'
▶ RESPONSE — 400 Bad Request
{
"error": "Student validation failed:
age: Cast to Number failed for
value \"twenty\" at path \"age\""
}
These 400 responses come from the catch block in the POST route: res.status(400).json({ error: err.message }). Mongoose throws the error; Express catches it and sends the message back to the client.

Promise Syntax vs async/await

All Mongoose methods return promises. The routes in server.js use async/await, but you will see both styles in documentation and existing code.

Promise — .then() / .catch()

📄 promise-style.js
  1. Student.findById("67f038494c8f9371af7b5cdf")
  2. .then((student) => {
  3. student.age = 25;
  4. return student.save();
  5. })
  6. .then((updated) => console.log("Updated:", updated))
  7. .catch((err) => console.error("Error:", err));
  • Works in all JavaScript environments
  • Gets deeply nested for multi-step operations
  • Errors handled by .catch()

async/await — used in our routes

📄 async-style.js
  1. async function updateStudent(id) {
  2. try {
  3. const student = await Student.findById(id);
  4. student.age = 25;
  5. const updated = await student.save();
  6. console.log("Updated:", updated);
  7. } catch (err) {
  8. console.error("Error:", err);
  9. }
  10. }
  12. updateStudent("67f038494c8f9371af7b5cdf");
  • Reads like synchronous code — much easier to follow
  • Stays flat even for multi-step operations
  • Errors caught by try/catch

Common Errors & Fixes

Error messageWhat it meansFix
ECONNREFUSED 127.0.0.1:27017 MongoDB server is not running Open a separate terminal and run mongod
Cannot find module './config/database' Wrong path in require(), or the file does not exist Confirm both files exist and paths are relative to server.js
Path `age` is required A required schema field is missing from the request body Include the field in the JSON you send, or make it optional in the schema
Cast to ObjectId failed The :id in the URL is not a valid 24-char MongoDB ObjectId Copy the full _id string from a GET response — do not type it by hand
Cannot read properties of null findById returned null and you accessed a field on it Always check if (!student) return res.status(404)... before using the result
PUT returns the old document findByIdAndUpdate returns the pre-update doc by default Pass { new: true } as the third argument

Takeaways

  • Mongoose adds schemas and validation — the raw driver silently accepts any data shape, Mongoose rejects bad data immediately
  • A schema defines what fields a document must have; a model compiles that schema into a class with built-in CRUD methods
  • Mongoose auto-maps "Student" → students collection (lowercase + pluralize)
  • All model methods return promises — use async/await + try/catch in route handlers
  • Always pass { new: true } to findByIdAndUpdate if you want the updated document back
  • Always use $set with update methods — omitting it replaces the entire document
  • Separate your code: config/database.js connects, models/ defines shapes, server.js runs the API
