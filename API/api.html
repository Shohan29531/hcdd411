Typical Web Application Structure

🖥️ Frontend (Client-side)

  • HTML
  • CSS
  • JavaScript
  • Frameworks: React / Angular / Vue

🧠 Backend (Server-side)

  • Processes requests
  • Performs business logic
  • Interacts with database (if needed)

🗄️ Database

  • Store data
  • Retrieve data
  • Update data
  • Delete data

Frontend ↔ Backend Communication

  • The frontend interacts with the backend using several methods
  • Depends on application architecture and requirements
  • Most commonly used method: API
Why APIs are common:
  • Ease of Use
  • Scalability
  • Separation of concerns
  • Standardized Communication
  • Security

API Key Components

  • Endpoint – specific URL (base URL + path)
  • HTTP Methods – GET / POST / PUT / DELETE
  • Request parameters / data – query string, body, etc.
  • Response – usually JSON or XML
  • Headers – e.g., Content-Type, auth tokens
  • Status codes – e.g., 200, 201, 400, 404, 500

XML vs JSON (Data Formats)

🧩 XML (Extensible Markup Language)

  • Structured format for storing/exchanging data
  • Uses tags
  • Supports string only (in this lecture context)
  • Slower to parse

⚡ JSON (JavaScript Object Notation)

  • Structured format for storing/exchanging data
  • Key-value pairs
  • Key must be string (double quotes)
  • Value supports: string, number, boolean, array, object, null
  • Faster and lightweight

JSON Object vs JSON String

  • JSON Object – parsed from a JSON string, usable as a data structure
  • Use JSON.stringify() to convert object → JSON string
  • JSON String – a string following JSON syntax rules
  • Use JSON.parse() to convert JSON string → object

Promises: Why .then() and .catch()?

  • jQuery AJAX uses callbacks: success / error
  • Axios and Fetch return a Promise
  • A Promise represents a value you will get later (success) or an error you will handle (failure)
Key idea: Promises avoid “callback nesting” and make async code easier to compose.
  • .then(...) = “when it succeeds, do this”
  • .catch(...) = “if it fails, do this”

Axios: Setup + Run (Step-by-step)

🌐 In the Browser

  • Add Axios via CDN (a script tag)
  • Then axios.get(...) works immediately
  • Great for small demos / learning

🧰 In Node.js

  • Install: npm i axios
  • Import: const axios = require("axios")
  • Run: node app.js

✅ Why students like Axios

  • Auto JSON parsing (response is already usable)
  • Response payload is in response.data
  • Cleaner POST syntax vs jQuery AJAX
Step-by-step (browser):
  1. Include Axios (CDN) before your script
  2. Call axios.get(url) → returns a Promise
  3. Use .then(...) (or await)
  4. Read the JSON from response.data

jQuery AJAX vs Axios: What changes?

Big picture:
  • jQuery AJAX: you pass options + callbacks (success/error)
  • Axios: you call a method (get/post/put/delete) and handle a Promise (.then/.catch)

Request body

  • jQuery: usually needs JSON.stringify(data)
  • Axios: just pass a JS object

Response

  • jQuery: response is already an object
  • Axios: response payload is in response.data

Errors

  • jQuery: error: function(err){...}
  • Axios: .catch(err => ...)

Fetch: Step-by-step (What happens?)

  • fetch(url) returns a Promise that resolves to a Response
  • The Response is metadata (status, headers) + a body stream
  • To read JSON, you must call response.json() (this is also async)
Common confusion: Fetch does NOT auto-parse JSON, and does not automatically throw on HTTP 404/500.
  • You should check response.ok and throw an error yourself

async / await: How to use it (Step-by-step)

  • async makes a function return a Promise
  • await pauses inside that async function until the Promise resolves
  • Use try/catch to handle errors like a normal synchronous program
Translation in your head:
  • await something() ≈ something().then(...) (same idea, easier to read)
  • But await only works inside an async function (in this lecture style)

FETCH API

  • Currently most popular in front-end web development
  • A built-in JavaScript API used to make HTTP requests
  • Modern + lightweight
  • Fetch API & Axios dominate modern web development
Important: Fetch requires manual JSON parsing:
  • .then((response) => response.json())
  • or const data = await response.json()
