async/await: Cleaner Promises

You already learned the core idea: a Promise means “result comes later.”
async/await is just a cleaner way to write Promise-based code (same concept, nicer syntax).

Translation table
  • .then(result => ...) ⟶ const result = await ...
  • .catch(err => ...) ⟶ try { ... } catch(err) { ... }
  • async ⟶ “this function returns a Promise”
  • await ⟶ “pause inside this async function until the Promise finishes”

Important detail

  • await pauses only the async function
  • The rest of your page/script keeps running
  • This is why UI does not “freeze”

Most common pattern

  • 1) await fetch(url)
  • 2) check response.ok
  • 3) await response.json()
  • 4) use the data

Axios: Optional Convenience Library

First: What is HTTP?
  • HTTP = the “rules” for how a client (browser) talks to a server
  • An HTTP request contains: method + URL + headers + (optional) body
  • An HTTP response contains: status code (200/404/500) + headers + body (often JSON)

Concrete example

  • Request: GET /users/1
  • Response: 200 OK + JSON body (user object)
  • If missing: 404 Not Found
  • If server breaks: 500 Server Error

Where Fetch/Axios fit

  • Fetch and Axios are just tools to send HTTP requests
  • They don’t change HTTP — they change how you write JS

Axios is an HTTP client library that returns Promises too (so you can use .then OR async/await).

Why the line “Axios is NOT a new async concept” is true:
  • The async concept is: “the result comes later” → that’s a Promise
  • Fetch returns a Promise
  • Axios returns a Promise
  • So Axios isn’t “new async” — it’s just another HTTP tool that uses Promises

✅ Why people like Axios

  • Auto JSON parsing (data is in response.data)
  • Cleaner syntax
  • Helpful error details
  • Interceptors (auth tokens)

Fetch vs Axios (key difference)

  • Fetch: you manually do await res.json()
  • Axios: JSON is already parsed in response.data
Bottom line:

HTTP stays the same. Your code style changes: callbacks → Promises → async/await.

So… Which One Should You Use?

✅ Class default recommendation:

Use Fetch + async/await for this course. It’s built-in, teaches the real fundamentals, and avoids extra libraries.

Pick Fetch when…

  • You want no dependencies
  • You want to learn the fundamentals
  • You’re writing typical browser API calls

Pick Axios when…

  • You want convenience (less boilerplate)
  • You need interceptors (auth tokens)
  • You want consistent error behavior

Avoid starting with…

  • $.ajax for new code
  • (Use it mainly for legacy jQuery projects)
Bottom line:

HTTP is the same. Only the JavaScript style changes: callbacks → Promises → async/await.
