API Fundamentals, JSON, and jQuery AJAX

Typical Web Application Structure

🖥️ Frontend (Client-side)

  • HTML
  • CSS
  • JavaScript
  • Frameworks: React / Angular / Vue

🧠 Backend (Server-side)

  • Processes requests
  • Performs business logic
  • Interacts with database

🗄️ Database

  • Store data
  • Retrieve data
  • Update data
  • Delete data

Frontend ↔ Backend Communication

  • The frontend interacts with the backend using several methods
  • Depends on application architecture and requirements
  • Most commonly used method: API
Why APIs are common:
  • Ease of Use
  • Scalability
  • Separation of concerns
  • Standardized Communication
  • Security

API Key Components

  • Endpoint – specific URL (base URL + path)
  • HTTP Methods – GET / POST / PUT / DELETE
  • Request parameters / data – query string, body, etc.
  • Response – usually JSON or XML
  • Headers – e.g., Content-Type, auth tokens
  • Status codes – e.g., 200, 201, 400, 404, 500

XML vs JSON (Data Formats)

🧩 XML

  • Uses tags like HTML
  • More verbose
  • Slower to parse

⚡ JSON

  • Key-value pairs
  • Lightweight
  • Fast to parse
  • Most commonly used!

JSON Object vs JSON String

  • JSON Object – usable as a JavaScript data structure
  • JSON String – text for sending over network
  • JSON.stringify() → object to string
  • JSON.parse() → string to object

jQuery AJAX

Syntax: $.ajax({})

  • url – endpoint URL
  • method – GET, POST, PUT, DELETE
  • data – Data to send (for POST/PUT) OR query params for GET
  • contentType – Use "application/json" for JSON request body
  • success – callback when request succeeds
  • error – callback when request fails
💡 Key Points:
  • jQuery auto-parses JSON responses (most common)
  • For POST/PUT with JSON body, use JSON.stringify()
  • data + GET → becomes query string (example: { userId: 1 } → ?userId=1)

AJAX Call Syntax Patterns (More Detail)

Three common styles you’ll see:
  • Callbacks: success, error inside $.ajax({ ... })
  • Promise-style: $.ajax(...).done(...).fail(...).always(...)
  • Shorthand helpers: $.get(...), $.post(...), $.getJSON(...)
Quick mapping:
  • contentType → what YOU are sending (request body type)
  • dataType → what you EXPECT back (response type, often "json")
  • headers → auth tokens, etc. (example: Authorization)
  • timeout → fail after X ms if server doesn’t respond
